# 05 数据集设计与标注规模

## 核心数据集定位

建议构建：

> Age-Friendly Indoor Spatial World Model Dataset

中文：

> 老龄友好室内空间世界模型数据集

它不应只是室内图片数据集，而应由以下层组成：

1. 室内空间几何数据
2. 室内 3D / 视觉数据
3. 家具与室内物体数据
4. 老龄友好与无障碍标签
5. 老人行为 / ADL / 路径数据
6. 改造建议与专家评价数据

## 自己最应该标注什么

不建议大量手工标注：

- 每个像素的 segmentation
- 每个家具的精确 3D mesh
- 大规模点云语义标注
- 从零建立 3D 模型
- 几千张图的 bounding box

这些可以依靠公开数据集和预训练模型。

最应该自己标注：

- 老龄友好评分
- 跌倒风险等级
- 通道可达性
- 护理操作空间
- 夜间行动安全
- 家具阻碍关系
- 改造建议
- 专家解释文本

## 数据单元设计

每个案例建议包含：

```text
Case ID
- 住宅类型：公寓 / 排屋 / 养老机构 / 农村住宅
- 使用者画像：年龄、行动能力、是否用轮椅/助行器、护理需求
- 空间数据：平面图、照片、360 图、点云、3D mesh
- 物体数据：家具类别、位置、尺寸、材质
- 空间关系：床-卫生间距离、门宽、通道宽度、家具阻挡关系
- 环境数据：照度、温湿度、噪音、空气质量
- 风险标签：跌倒风险、无障碍风险、护理困难、认知障碍风险
- 专家评分：老龄友好评分、可达性评分、安全评分、舒适评分
- 改造建议：布局调整、家具替换、扶手、照明、地面处理
```

## 标注规模建议

### 基于相近研究的数据规模校准

可参考的知名研究规模大致分三类：

| 研究类型 | 代表数据/研究 | 规模 | 对本研究的启发 |
|---|---|---:|---|
| 大型室内 3D 视觉数据集 | ScanNet | 1513 个室内 scans，约 250 万 RGB-D views | 说明 3D scene understanding 的基础视觉能力通常依赖大规模公开数据，不应完全自己采集 |
| 建筑级 RGB-D 数据 | Matterport3D | 90 个 building-scale scenes，10800 个全景视图 | 说明真实建筑级 3D 数据几十到上百栋已可支撑重要 benchmark |
| 合成/结构化室内数据 | Structured3D | 3500 个 scenes，21835 个 rooms | 说明大规模结构化空间训练可依赖合成数据 |
| 家具布局/室内场景生成 | 3D-FRONT | 6813 个 CAD houses，约 18968 个 furnished rooms | 说明布局生成需要大量房间，但可以借助公开/合成数据 |
| 平面图解析 | CubiCasa5K | 5000 个 floorplan samples，80+ 类标注 | 说明平面图识别类任务的可信公开规模约为数千张 |
| 室内无障碍/安全 AI 原型 | RASSAR | 早期版本单个 home，CHI 2024 扩展到 10 个 homes 技术评估 | 说明 accessibility/safety 这种专家规则 + CV 的方向，早期 proof-of-concept 不需要上千真实住宅 |
| 家庭跌倒风险评估工具 | HOME FAST / Thai-HFHAT 预测效度研究 | 450 名 60+ older adults，12 个月随访 | 说明如果研究目标是预测真实跌倒结果，样本量需要达到数百人/户以上 |
| 专家内容效度研究 | home fall hazard checklist | 9-16 名专家评审 74 项 checklist | 说明专家标签体系本身可以先用 5-10+ 名专家建立，不一定先标几千空间 |

因此，你的博士研究不应该把所有数据都自己采。更合理的是：

> 公开室内视觉/3D/平面图数据集负责模型基础能力，自己标注的重点是养老、适老、无障碍、跌倒风险、护理空间和改造建议。

### 最低可跑通效果

| 数据类型 | 建议最低数量 | 能做到什么 |
|---|---:|---|
| 真实住宅/养老空间案例 | 20-30 套住宅/机构空间，约 80-150 个房间 | 真实验证集和 proof-of-concept |
| 公开数据中重新标注的房间 | 300-500 个房间 | 初步风险评估模型 |
| 专家改造建议案例 | 100-200 个房间 | 初步训练/评估生成式建议 |
| 详细 3D/点云/全景样本 | 30-80 个空间 | 3D scene understanding demo |
| 老龄友好评分标签 | 500-800 个房间级标签 | 分类/评分模型 |

这个规模可以看到基本效果，例如上传图片、平面图或 3D 扫描后，模型能判断通道过窄、家具阻挡、床到卫生间路径不安全、照明不足、扶手缺失等问题。

如果低于这个规模，例如只有 50-100 个房间，比较适合做概念验证和访谈研究，不适合声称训练出稳定模型。

### 适合发论文的规模

| 层级 | 建议数量 |
|---|---:|
| 房间级样本 | 1000-2000 个房间 |
| 住宅级真实案例 | 50-100 套住宅/养老空间 |
| 专家评分 | 每个房间 2-3 名专家中的至少 1-2 名标注 |
| 高质量详细标注 | 200-500 个房间 |
| 测试集 | 至少 100-200 个房间，最好包含真实住宅 |

这是最推荐写进博士 proposal 的规模。它足够像一个可信数据集，又不会像大型 CV benchmark 那样不可执行。

### 强 benchmark 规模

| 数据类型 | 数量 |
|---|---:|
| 总房间数 | 3000-5000+ |
| 真实住宅/养老空间 | 150-300 套 |
| 专家改造建议 | 1000+ 条 |
| 3D/点云/全景数据 | 500+ 个空间 |
| 老龄友好标签 | 5000+ 条 room-level / object-level labels |

## 最现实的 PhD 数据路线

### 第一阶段

从公开数据集选 500 个房间：

- 卧室
- 卫生间
- 厨房
- 客厅

标注：

- 通道是否足够
- 是否有跌倒风险
- 家具是否阻挡
- 夜间路径是否安全
- 是否适合轮椅/助行器
- 是否需要扶手/照明/家具调整
- 老龄友好评分 1-5 分

### 第二阶段

采集 30-50 套真实住宅：

- 普通手机照片
- 全景图
- 平面图
- 简单尺寸测量
- 若条件允许，使用手机 LiDAR 或深度相机

### 第三阶段

找 2-3 位专家标注部分样本：

- 不需要全部多人标注。
- 至少 20%-30% 的样本需要重复标注。
- 用于计算一致性，例如 Cohen's kappa 或 ICC。

### 第四阶段

用预训练模型做 baseline：

- CLIP
- LLaVA
- SAM
- YOLO
- Depth Anything
- ScanNet / Matterport3D / Structured3D 预训练模型

## 推荐 proposal 写法

建议写：

> 本研究计划构建一个包含约 1000-2000 个室内房间样本的老龄友好空间标注数据集，其中包括 50-100 套真实住宅/养老空间案例，并结合公开 3D 室内数据集进行预训练与验证。

实际执行时先做：

> 500 个房间 + 20-30 套真实案例

这是最现实、也足够看到基本模型效果的规模。

## 最终样本量建议

最低可行版本：

> 300-500 个公开房间的适老标签 + 20-30 套真实住宅/养老空间 + 100-200 条专家改造建议。

博士 proposal 最佳版本：

> 1000-2000 个房间级样本 + 50-100 套真实住宅/养老空间 + 200-500 个高质量多模态样本 + 2-3 名专家对核心测试集重复标注。

如果目标是做真实跌倒风险预测，而不仅是空间风险评估：

> 需要 300-500+ 名老人/住宅，并最好有 6-12 个月随访数据。

如果目标是 AI 空间理解和适老改造建议，则不必一开始做到随访级别；可以先做空间风险标签和专家评价。
