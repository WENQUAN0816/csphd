# 05 数据集设计与标注规模

## 核心数据集定位

建议构建：

> Age-Friendly Indoor Spatial World Model Dataset

中文：

> 老龄友好室内空间世界模型数据集

它不应只是室内图片数据集，而应由以下层组成：

1. 室内空间几何数据
2. 室内 3D / 视觉数据
3. 家具与室内物体数据
4. 老龄友好与无障碍标签
5. 老人行为 / ADL / 路径数据
6. 改造建议与专家评价数据

## 自己最应该标注什么

不建议大量手工标注：

- 每个像素的 segmentation
- 每个家具的精确 3D mesh
- 大规模点云语义标注
- 从零建立 3D 模型
- 几千张图的 bounding box

这些可以依靠公开数据集和预训练模型。

最应该自己标注：

- 老龄友好评分
- 跌倒风险等级
- 通道可达性
- 护理操作空间
- 夜间行动安全
- 家具阻碍关系
- 改造建议
- 专家解释文本

## 数据单元设计

每个案例建议包含：

```text
Case ID
- 住宅类型：公寓 / 排屋 / 养老机构 / 农村住宅
- 使用者画像：年龄、行动能力、是否用轮椅/助行器、护理需求
- 空间数据：平面图、照片、360 图、点云、3D mesh
- 物体数据：家具类别、位置、尺寸、材质
- 空间关系：床-卫生间距离、门宽、通道宽度、家具阻挡关系
- 环境数据：照度、温湿度、噪音、空气质量
- 风险标签：跌倒风险、无障碍风险、护理困难、认知障碍风险
- 专家评分：老龄友好评分、可达性评分、安全评分、舒适评分
- 改造建议：布局调整、家具替换、扶手、照明、地面处理
```

## 标注规模建议

### 最低可跑通效果

| 数据类型 | 建议最低数量 | 能做到什么 |
|---|---:|---|
| 真实住宅/房间案例 | 30-50 套住宅，约 150-300 个房间 | 真实验证集 |
| 公开数据中重新标注的房间 | 300-500 个房间 | 初步风险评估模型 |
| 专家改造建议案例 | 100-200 个房间 | 训练/评估生成式建议 |
| 详细 3D/点云标注 | 30-80 个空间 | 3D scene understanding demo |
| 老龄友好评分标签 | 500-1000 个房间级标签 | 分类/评分模型 |

这个规模可以看到基本效果，例如上传图片、平面图或 3D 扫描后，模型能判断通道过窄、家具阻挡、床到卫生间路径不安全、照明不足、扶手缺失等问题。

### 适合发论文的规模

| 层级 | 建议数量 |
|---|---:|
| 房间级样本 | 1000-2000 个房间 |
| 住宅级真实案例 | 50-100 套 |
| 专家评分 | 每个房间 2-3 名专家中的至少 1-2 名标注 |
| 高质量详细标注 | 200-500 个房间 |
| 测试集 | 至少 100-200 个房间，最好包含真实住宅 |

### 强 benchmark 规模

| 数据类型 | 数量 |
|---|---:|
| 总房间数 | 3000-5000+ |
| 真实住宅/养老空间 | 150-300 套 |
| 专家改造建议 | 1000+ 条 |
| 3D/点云/全景数据 | 500+ 个空间 |
| 老龄友好标签 | 5000+ 条 room-level / object-level labels |

## 最现实的 PhD 数据路线

### 第一阶段

从公开数据集选 500 个房间：

- 卧室
- 卫生间
- 厨房
- 客厅

标注：

- 通道是否足够
- 是否有跌倒风险
- 家具是否阻挡
- 夜间路径是否安全
- 是否适合轮椅/助行器
- 是否需要扶手/照明/家具调整
- 老龄友好评分 1-5 分

### 第二阶段

采集 30-50 套真实住宅：

- 普通手机照片
- 全景图
- 平面图
- 简单尺寸测量
- 若条件允许，使用手机 LiDAR 或深度相机

### 第三阶段

找 2-3 位专家标注部分样本：

- 不需要全部多人标注。
- 至少 20%-30% 的样本需要重复标注。
- 用于计算一致性，例如 Cohen's kappa 或 ICC。

### 第四阶段

用预训练模型做 baseline：

- CLIP
- LLaVA
- SAM
- YOLO
- Depth Anything
- ScanNet / Matterport3D / Structured3D 预训练模型

## 推荐 proposal 写法

建议写：

> 本研究计划构建一个包含约 1000-2000 个室内房间样本的老龄友好空间标注数据集，其中包括 50-100 套真实住宅/养老空间案例，并结合公开 3D 室内数据集进行预训练与验证。

实际执行时先做：

> 500 个房间 + 30 套真实案例

这是最现实、也足够看到基本模型效果的规模。
